---
name: Autonomous repository work
alwaysApply: true
---

Explore autonomamente o projeto aberto antes de responder.

Use as ferramentas disponíveis para localizar e ler arquivos, instruções,
documentação, testes, logs, diffs e histórico relevantes. Execute os comandos
necessários para investigar e validar a tarefa.

Nunca peça ao usuário para fornecer conteúdo que esteja no repositório ou que
possa ser obtido pelo terminal. Só faça perguntas quando a informação não puder
ser descoberta no ambiente.

## Disciplina de execução

- Cumpra exatamente o tipo de tarefa solicitado.
- Quando o pedido for analisar ou propor, não edite arquivos.
- Não presuma a existência de arquivos pelo nome; localize-os com as ferramentas.
- Antes de declarar o estado de uma funcionalidade, verifique documentação e código.
- Não confunda arquivos de configuração das ferramentas de desenvolvimento com
  arquivos pertencentes ao projeto.
- Se uma operação falhar, investigue a causa e procure alternativas de leitura
  antes de concluir que o ambiente proíbe a operação.
  - “Analisar”, “revisar”, “explicar” e “propor” são tarefas somente de leitura.
- Só edite arquivos quando o usuário pedir explicitamente para implementar,
  corrigir ou alterar.
- Antes de implementar, leia os testes e todos os contratos diretamente
  relacionados.
- Nunca declare uma implementação validada sem executar os comandos relevantes.
- Se o terminal estiver disponível, solicite autorização para executar os
  comandos; não afirme que não pode executá-los sem tentar.
